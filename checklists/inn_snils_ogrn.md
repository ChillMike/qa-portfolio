# Чек-лист: валидация госданных (ИНН/СНИЛС/ОГРН)

> Использовать перед каждым релизом, затрагивающим форму ввода госданных.

## ✅ ИНН физлица (12 цифр)

- [ ] 12 цифр, КС=0 → валиден (пример: `770708389309`)
- [ ] 12 цифр, КС=10 → **ошибка** (пример: `770708389310`)
- [ ] 11 цифр → ошибка «12 цифр»
- [ ] 13 цифр → ошибка «12 цифр»
- [ ] Спецсимволы (`@`, `#`, `_`) → ошибка «только цифры»
- [ ] Кириллица → ошибка «только цифры»
- [ ] Пробелы/дефисы → автоудаление → валиден
- [ ] Все нули (`000000000000`) → ошибка
- [ ] Все девятки (`999999999999`) → ошибка
- [ ] Отправка в ФНС только для валидных ИНН (КС≠10)

## ✅ ИНН юрлица (10 цифр)

- [ ] 10 цифр, КС=0 → валиден (пример: `7707083893`)
- [ ] 10 цифр, КС=10 → **ошибка**
- [ ] 9 цифр → ошибка «10 цифр»
- [ ] 11 цифр → ошибка «10 цифр»
- [ ] Спецсимволы → ошибка «только цифры»

## ✅ СНИЛС (11 цифр)

- [ ] Формат `XXX-XXX-XXX-YY` → валиден
- [ ] Без дефисов (`11223344590`) → автоформатирование
- [ ] КС=100 → валиден (максимум)
- [ ] КС=101 → ошибка
- [ ] Спецсимволы → ошибка «только цифры и дефисы»
- [ ] Отправка в ПФР только для валидных СНИЛС

## ✅ ОГРН (13 цифр для юрлиц)

- [ ] 13 цифр → валиден (пример: `1027739212345`)
- [ ] 12 цифр → ошибка «13 цифр»
- [ ] 14 цифр → ошибка «13 цифр»
- [ ] Алгоритм валидации: остаток от деления первых 12 цифр на 11 → должен совпадать с 13-й цифрой
- [ ] Спецсимволы → ошибка «только цифры»

## ✅ Общие проверки

- [ ] Сообщения об ошибках понятны пользователю (не «Validation failed»)
- [ ] Курсор фокусируется на поле с ошибкой
- [ ] Подсветка поля красным цветом
- [ ] Отправка формы заблокирована при наличии ошибок
- [ ] Логирование попыток ввода невалидных данных (для анализа)
