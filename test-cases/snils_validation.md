# Тест-кейсы: валидация СНИЛС (11 цифр)

> Формат: `XXX-XXX-XXX-YY` (где YY — контрольное число). Алгоритм валидации ПФР.

## Граничные значения контрольного числа

| ID | Входные данные | Контрольное число | Ожидаемый результат | Приоритет |
|----|----------------|-------------------|---------------------|-----------|
| SNILS-01 | `112-233-445 90` | 90 | Валиден | Критичный |
| SNILS-02 | `112-233-445 00` | 00 | Валиден | Критичный |
| SNILS-03 | `112-233-445 100`| 100 | Ошибка: «Неверный СНИЛС» | Критичный |
| SNILS-04 | `001-001-998 88` | 88 | Валиден | Высокий |
| SNILS-05 | `001-001-998 89` | 89 | Ошибка: «Неверный СНИЛС» | Высокий |

## Форматирование и спецсимволы

| ID | Входные данные | Ожидаемый результат | Приоритет |
|----|----------------|---------------------|-----------|
| SNILS-06 | `11223344590` | Автоформатирование → `112-233-445 90` | Низкий |
| SNILS-07 | `112 233 445 90` | Автоформатирование → `112-233-445 90` | Низкий |
| SNILS-08 | `112-233-445-90` | Принят как валидный | Низкий |
| SNILS-09 | `112-233-445_90` | Ошибка: «Только цифры и дефисы» | Высокий |
| SNILS-10 | `СНИЛС11223344590` | Ошибка: «Только цифры» | Высокий |

## Интеграция с ПФР

| ID | Сценарий | Ожидаемый результат |
|----|----------|---------------------|
| SNILS-11 | Валидный СНИЛС + таймаут ПФР | Спиннер 30 сек → сообщение об ошибке → кнопка «Повторить» |
| SNILS-12 | Невалидный СНИЛС (КС=101) | Ошибка на клиенте **до** отправки в ПФР |
| SNILS-13 | СНИЛС с КС=100 | Валиден (максимальное допустимое значение) |
